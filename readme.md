# AI 軍師 · 對話包定價模擬器 v3 改版需求

## 專案背景

目前已有一個「AI 軍師 · 對話包定價模擬器 v2」的網頁工具（純前端，HTML/CSS/JS），包含三種計價模式（方案計價、點數計價、訂閱制），涵蓋 6 大維度：API 成本 → 對話包設計 → 營運成本 → 用戶流失 → 伺服器擴容 → 續購行為。

經過 PM 層級的完整審查，發現模擬器存在多項結構性盲點和缺失，需要進行 v3 改版。以下是完整的改版需求，請依照優先順序實作。

---

## 一、新增模組：⑦ 獲客成本與行銷漏斗（最高優先）

這是目前模型最大的缺陷——完全沒有計算獲取用戶需要花多少錢。

### 需要新增的欄位：
1. **月行銷預算** NT$（輸入框，快捷按鈕：1萬/3萬/5萬/10萬/20萬）
2. **平均每位訪客獲取成本（CPA/CPC）** NT$（輸入框，快捷按鈕：5/10/20/50）
   - 提示文字：「Google/FB 廣告平均 CPC NT$5-30，精準投放可能更高」
3. **Landing Page → 試用轉換率** %（滑桿+快捷按鈕：2%/5%/10%/15%/20%）
   - 提示文字：「SaaS 平均 Landing Page 轉換率 3-10%」
4. **試用 → 付費轉換率** %（此欄位訂閱制模式已有，方案計價和點數計價模式需同步加入）
   - 提示文字：「SaaS 免費試用轉付費通常 2-15%」

### 計算邏輯：
- **每月可觸及訪客數** = 月行銷預算 ÷ CPA
- **每月試用註冊數** = 訪客數 × Landing Page 轉換率
- **每月新增付費用戶** = 試用註冊數 × 試用轉付費率
- **CAC（單位獲客成本）** = 月行銷預算 ÷ 每月新增付費用戶
- **試用用戶 API 成本**（免費試用輪數 × 單輪成本 × 試用註冊數）要計入當月成本

### 顯示 KPI：
- CAC NT$___
- LTV（客戶終身價值）= ARPU × (1 ÷ 月流失率) × 毛利率
- **LTV / CAC 比率**（用顏色標示：< 1 紅色、1-3 橘色、> 3 綠色）
- Payback Period（回收獲客成本的月數）= CAC ÷ 每用戶月毛利

### 對現有邏輯的影響：
- 「月行銷預算」要計入每月的固定成本中
- 逐月明細表新增「行銷費」欄位
- 「初始付費用戶」欄位改為根據行銷漏斗自動計算（但保留手動覆寫的選項）
- 用戶成長率的 Q1-Q4 設定應與行銷漏斗計算連動，自動顯示「以目前行銷預算，能支撐的最大月成長率」

## 二、新增模組：⑧ 額外營運成本明細（高優先）

目前「每月基礎固定成本」是一個單一數字，過度簡化。需要拆解成明細。

### 需要新增的欄位（可折疊區塊）：
1. **人事成本** NT$/月（輸入框，快捷按鈕：0/5萬/10萬/20萬/40萬）
   - 提示文字：「工程師 6-12 萬、行銷 4-6 萬、客服 3-5 萬」
2. **辦公/雜支** NT$/月（輸入框，快捷按鈕：0/5000/1萬/2萬）
3. **第三方工具訂閱** NT$/月（輸入框，快捷按鈕：0/3000/5000/1萬）
   - 提示文字：「Slack、Notion、Analytics、監控工具等」
4. **支付平台手續費** %（輸入框，預設 2.8%，快捷按鈕：2%/2.8%/3.5%）
   - 提示文字：「綠界 2.75%、Stripe 2.9%+NT$9、LINE Pay 3%」
5. **預估退款率** %（輸入框，預設 2%，快捷按鈕：0%/2%/5%/10%）
   - 提示文字：「台灣消保法 7 天鑑賞期，SaaS 退款率通常 2-5%」
6. **營業稅** %（預設 5%，不可編輯但可顯示）

### 計算邏輯：
- 每月固定成本 = 人事 + 辦公雜支 + 第三方工具 + 行銷預算（來自⑦）
- 每月變動成本 = 營收 × 支付手續費% + 營收 × 退款率%
- 原本的「固定+GPU」欄位改為「營運總成本」= 固定成本 + 變動成本 + GPU 成本

### 逐月明細表更新：
- 「固定+GPU」欄位拆分為：「固定成本」「行銷費」「手續費+退款」「GPU」，或至少在 hover/展開時顯示明細

## 三、新增功能：情境比較與壓力測試（高優先）

### A. 三種計價模式並列比較表
在頁面底部（或可切換的獨立 tab），新增一個「📊 三模式比較」的檢視，將方案計價、點數計價、訂閱制的關鍵 KPI 並列顯示：

| 指標 | 方案計價 | 點數計價 | 訂閱制 |
|---|---|---|---|
| ARPU（每用戶月營收）| | | |
| 毛利率 | | | |
| CAC | | | |
| LTV | | | |
| LTV/CAC | | | |
| 損平衡用戶數 | | | |
| 回本月份 | | | |
| 前3月累計損益 | | | |
| 12月累計損益 | | | |

此比較表從三個模式的現有參數即時計算，讓使用者不需要手動切換 tab 就能做比較。

### B. 樂觀 / 基準 / 悲觀 三情境模擬
在每個計價模式內，新增一個「🔀 情境切換」按鈕組（樂觀 / 基準 / 悲觀），點擊後自動套用不同的參數組合：

**悲觀情境（預設偏移）：**
- 初始用戶 = 基準值 × 0.5
- 成長率 = 基準值 × 0.5
- 流失率 = 基準值 × 1.5
- 試用轉付費率 = 基準值 × 0.6
- 固定成本 = 基準值 × 1.3

**樂觀情境（預設偏移）：**
- 初始用戶 = 基準值 × 1.5
- 成長率 = 基準值 × 1.3
- 流失率 = 基準值 × 0.7
- 試用轉付費率 = 基準值 × 1.3
- 固定成本 = 基準值 × 0.8

使用者可以自訂偏移比例。三種情境的 12 月累計損益曲線疊在同一張圖上（用不同顏色/線型區分），讓使用者一眼看到最好和最壞的可能。

### C. 敏感性分析
新增一個「🎯 敏感性分析」區塊，讓使用者選擇一個核心變數（流失率/成長率/售價/CAC），模擬器自動計算該變數在 ±20%、±40% 變化時，回本月份和 12 月累計損益的變化。以表格或小型圖表呈現。

## 四、改進現有邏輯（中優先）

### A. 流失率改為 Cohort 留存曲線
目前流失率是固定常數（例如 5%/月），但真實情況是新用戶第 1-2 月流失率很高，留存越久的用戶越穩定。

**改法：**
- 新增「首月流失率」（預設 15-20%）和「穩定流失率」（預設 3-5%）兩個參數
- 計算邏輯改為：第 1 月用首月流失率，第 2 月用 (首月+穩定)/2 的過渡值，第 3 月起用穩定流失率
- 原本的單一流失率滑桿改為「簡易模式」（向下相容），新增「進階模式」切換

### B. 方案計價模式：自動提示最低不虧損售價
目前方案計價的預設值（200 輪 / NT$199）是嚴重虧損的，容易誤導使用者。

**改法：**
- 在每個方案卡片中，當售價 < API 成本時（毛利為負），用紅色警示框顯示：「⚠️ 此售價低於成本，建議最低售價 NT$___（含 30% 毛利）」
- 最低建議售價 = API 成本 ÷ (1 - 目標毛利率)，目標毛利率預設 30%
- 重新設定方案計價的預設值，確保預設狀態下所有方案都是正毛利

### C. 用戶降級行為（訂閱制適用）
目前假設用戶訂哪個方案就一直訂同方案。

**改法：**
- 新增「月降級率」%（預設 3%，範圍 0-10%）
- 提示文字：「每月有多少 % 的高階用戶降級到較低方案」
- 降級用戶的 ARPU 按降一階方案的月費計算

### D. GPU 成本邏輯釐清
目前 GPU 伺服器區塊的成本顯示 NT$0/月，但又顯示需要 4-6 台伺服器，邏輯矛盾。

**改法：**
- 新增一個 toggle：「技術架構選擇」
  - 選項 A：「使用第三方 API（OpenAI/Claude/Gemini）」→ 隱藏 GPU 區塊，API 成本已包含運算費
  - 選項 B：「自建推理伺服器」→ 顯示 GPU 區塊，正常計算伺服器成本
- 這樣使用者就不會搞混「到底需不需要算 GPU 成本」

## 五、UI/UX 改進（中優先）

### A. 逐月明細表強化
- 新增「累計營收」欄位
- 新增可展開的每月成本明細（點擊展開：API 成本、固定成本、行銷費、手續費、GPU）
- 表格支援 CSV 匯出功能（一個「📥 匯出 CSV」按鈕）

### B. 關鍵假設摘要卡
在頁面最頂部（方案設計區上方），新增一個「📋 關鍵假設摘要」的可折疊區塊，列出目前所有的核心假設值，讓使用者一眼掌握：
- 單輪 API 成本、ARPU、毛利率、CAC、LTV/CAC、流失率、成長率、固定成本總計
- 用紅/黃/綠色標示各指標的健康狀態

### C. 預設值智慧化
- 當使用者修改「單輪 API 成本」時，自動調整各方案的「建議售價」（但不強制覆蓋，只是在旁邊用灰色小字提示）
- 快捷按鈕增加 tooltip，解釋為什麼是這個數字（例如「10萬」按鈕 hover 顯示「適合 1 工程師 + 1 行銷的精實團隊」）

### D. 三模式參數同步選項
新增一個全域設定 toggle：「🔗 同步營運參數」
- 開啟時，三種計價模式共用相同的「營運基礎參數」「用戶流失」「成長率」「GPU 設定」
- 關閉時（預設），三種模式各自獨立設定
- 這讓使用者可以在「只改計價方式，其他條件相同」的情況下做公平比較

## 六、新增模組：⑨ 市場天花板與競品參考（低優先）

### 欄位：
1. **目標市場規模（TAM）** 人數（輸入框，快捷按鈕：5000/1萬/5萬/10萬/50萬）
   - 提示文字：「台灣願意每月付 NT$500+ 的 AI 工具用戶估計 5-20 萬人」
2. **你的可觸及市佔率（SOM）** %（滑桿，預設 1-5%）

### 計算邏輯：
- 最大可觸及用戶數 = TAM × SOM%
- 當模擬的第 N 月活躍用戶數超過此上限時，成長率自動歸零，並顯示警示：「⚠️ 第 N 月已觸及市場天花板，成長將趨緩」
- 在 12 月損益圖表上畫一條水平虛線標示天花板

### 競品參考（靜態資訊卡）：
在方案設計區旁邊，新增一個可折疊的「📊 市場定價參考」小卡，列出主流 AI 工具的定價：
- ChatGPT：免費 / Plus NT$650 / Pro NT$3,300
- Claude：免費 / Pro NT$650 / Max NT$3,300
- Gemini：免費 / Advanced NT$650
- 提示文字：「你的定價需要與這些競品有明確的價值差異」

## 七、技術與實作注意事項

1. **資料儲存**：維持 localStorage 自動儲存，但 v3 新增的欄位也要一併儲存。注意版本遷移——舊版使用者打開 v3 時，新欄位使用預設值，不能 crash。
2. **響應式設計**：目前已有的 RWD 請維持，新增區塊也要在手機上可用。
3. **效能**：所有計算都在前端即時完成，修改任何參數時，底部的圖表和表格要即時更新（目前已有此行為，請維持）。
4. **程式碼結構**：建議將計算邏輯拆成獨立的 JS module（如 `calculator.js`），UI 渲染拆成 `renderer.js`，方便未來維護。
5. **語言**：全站使用繁體中文，數字格式使用 NT$ 和千分位逗號。
6. **圖表**：現有圖表用的是什麼 library 請維持，新增的圖表（情境疊合圖、敏感性分析圖）使用同一套。
